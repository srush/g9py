g9.py
automatically interactive graphics - in python?

Welcome to the mini-g9 gallery! Drag all the graphics!

This is a shameless copy of g9.js, written in Python. The goal here is to show off some of the cool things you can do with Pyodide for running demos in the browser.

The prototype idea is that we write our graphics as standard PyTorch parameterized functions ("neural networks"). Instead of optimizing them for a loss, we instead optimize them to match the movement of the users cursor. In order to update the graphics we use backpropagation (implemented through a pure-python torch library minitorch). In theory any parameterized function could be used this way.

(The examples and text here is adapted from the g9.js examples page.)

Basic

A minimal example with only two points. Our render function always draws one point at (x, y), and the other point at (y, x), so when you drag one of the points, the other kind of mirrors it.
